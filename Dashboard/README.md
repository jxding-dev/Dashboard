# 작업자 대시보드

개인 작업 관리, 프롬프트 정리, 로드맵 계획, 수익 계산을 한 화면에서 다룰 수 있는 정적 웹 대시보드입니다.  
별도 백엔드 없이 `HTML + CSS + Vanilla JavaScript`로 구성되어 있으며, 데이터는 브라우저 `localStorage`에 저장됩니다.

## 개요

이 프로젝트는 1인 작업자나 프리랜서가 다음 정보를 한 곳에서 관리할 수 있도록 만든 대시보드입니다.

- 체크리스트 진행 상황
- 월별 로드맵과 실행 메모
- 프롬프트 템플릿 보관
- 작업 플로우 정리
- 플랫폼/채널 정보 정리
- 예상 수익 계산 및 기록

대시보드는 사이드바 네비게이션으로 각 화면을 전환하며, 주요 수치는 대시보드 첫 화면에서 요약 카드로 확인할 수 있습니다.

## 주요 기능

### 1. 대시보드

- 체크리스트 완료 수, 예상 순수익, 로드맵 개수, 프롬프트 개수를 요약 카드로 표시
- 카드 드래그 앤 드롭으로 순서 변경
- 대시보드 메모 저장
- 프로세스 카드 추가 및 관리

### 2. 체크리스트

- 그룹 단위 체크리스트 생성
- 항목 추가, 완료 처리, 삭제
- `긴급`, `오늘`, `이번 주` 태그 관리
- 전체 진행률 자동 계산

### 3. 로드맵

- 월 단위 목표 작성
- 월별 작업 목록 추가/삭제
- 메모 기록

### 4. 레이아웃 가이드

- 섹션별 이름, 팁, 크기 정보 기록
- 작업 화면이나 콘텐츠 구성안을 메모처럼 정리 가능

### 5. 프롬프트 뱅크

- 프롬프트 추가, 수정, 삭제
- 카테고리 분류
  - `Claude`
  - `ChatGPT`
  - `Gemini`
  - `Photoshop`
  - `기타`
- 검색 및 필터 기능
- 클립보드 복사 기능

### 6. 워크플로우

- 작업 단계를 툴 기준으로 정리
- 단계별 제목/설명 기록
- 운영 메모 저장

### 7. 플랫폼 관리

- 플랫폼명, 유형, 단가, 전략, 상태 기록
- 상태 관리
  - 활성
  - 준비중
  - 보류

### 8. 수익 계산기

- 단가, 월 프로젝트 수, 비용, 세율 입력
- 총매출, 세금, 비용, 예상 순수익 자동 계산
- 계산 결과 저장 및 이력 확인

### 9. 데이터 백업/복원

- 현재 데이터를 JSON 파일로 내보내기
- 백업 JSON 파일 다시 불러오기
- 전체 데이터 초기화

## 기술 스택

- `HTML5`
- `CSS3`
- `Vanilla JavaScript`
- 브라우저 `localStorage`

## 실행 방법

별도 빌드 과정이 없는 정적 웹 프로젝트입니다.

### 가장 간단한 실행

1. 프로젝트 폴더로 이동합니다.
2. `index.html` 파일을 브라우저에서 직접 엽니다.

### 권장 실행

브라우저 보안 정책이나 파일 접근 이슈를 줄이려면 간단한 로컬 서버로 실행하는 편이 좋습니다.

예시:

```powershell
cd E:\practice\작업자대시보드
python -m http.server 8000
```

이후 브라우저에서 아래 주소를 엽니다.

```text
http://localhost:8000
```

## 데이터 저장 방식

- 앱 데이터는 브라우저 `localStorage`의 `mymanager_v1` 키에 저장됩니다.
- 브라우저 캐시/저장소를 삭제하면 데이터가 사라질 수 있습니다.
- 중요한 데이터는 반드시 내보내기 기능으로 JSON 백업을 권장합니다.

## 프로젝트 구조

```text
작업자대시보드/
├─ index.html            # 전체 화면 구조
├─ style.css             # 스타일 정의
├─ config.js             # 상수, 기본 데이터, 설정값
├─ utils.js              # 공통 유틸리티, sanitize 함수
├─ store.js              # localStorage 저장/불러오기, 백업/복원
├─ ui.js                 # 토스트, 모달, 페이지 전환, 테마
├─ feature-planning.js   # 대시보드, 체크리스트, 로드맵
├─ feature-tools.js      # 레이아웃 가이드, 프롬프트, 워크플로우
├─ feature-finance.js    # 플랫폼 관리, 수익 계산기
└─ app.js                # 초기화 및 이벤트 연결
```

## 사용 시 참고

- 별도 회원 기능은 없으며, 현재 사용 중인 브라우저에만 데이터가 저장됩니다.
- 다른 PC나 브라우저로 옮길 때는 JSON 내보내기 후 가져오기를 사용해야 합니다.
- 프로젝트 문구 중 일부는 한글 인코딩 환경에 따라 깨져 보일 수 있으므로, 파일은 `UTF-8` 기준으로 관리하는 것을 권장합니다.
